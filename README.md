# Tinterest

Внутреннее корпоративное приложение Т-Банка для знакомства сотрудников по общим интересам и хобби. Позволяет находить коллег с похожими увлечениями, общаться в чате и вступать в тематические группы.

## Возможности

- **Матчинг** — алгоритм подбора коллег по анкете интересов с отображением процента совместимости
- **Личные чаты** — создаются автоматически после подтверждения матча, работают через WebSocket
- **Групповые чаты** — тематические группы по интересам с возможностью создания своих
- **Профиль** — имя, фото, должность, город, отдел, список интересов
- **Административная панель** — управление пользователями и просмотр ключевых метрик

## Быстрый старт

### 1. Клонировать репозиторий

```bash
git clone <url репозитория>
cd <папка проекта>
```

### 2. Создать файл с переменными окружения

```bash
cp .env.example .env
```

Открыть `.env` и заполнить значения

### 3. Создать папку `secrets/` и положить в нее ключ .p12 для https и rsa ключи для подписи и проверки jwt

Положить её в корень проекта. Структура должна быть такой:

```
secrets/
├── keystore.p12
└── keys/
    ├── private.pem
    └── public.pem
```

### 4. Запустить

```bash
docker-compose up --build
```

Backend будет доступен по адресу: **https://localhost:8443**

## Тестирование

Проект использует [Allure Report](https://allurereport.org/) для генерации отчётов по автотестам

### Запуск тестов

```bash
mvn test
```

После выполнения тестов результаты будут сохранены в:

```
target/allure-results
```

### Генерация и открытие отчётов

```bash
allure serve target/allure-results
```

После выполнения команды Allure автоматически:

- сгенерирует HTML-отчёт;
- запустит локальный сервер;
- откроет отчёт в браузере.

## Интеграция с Github actions

Проект настроен на автоматический запуск тестов и публикацию Allure-отчёта при каждом push или pull request в ветку `main`.

### Как это работает

Pipeline состоит из двух джоб:

**`test`** — сборка и запуск тестов:
- Поднимает JDK 17
- Запускает `mvn test`
- Сохраняет результаты (`target/allure-results`) как артефакт

**`reports`** — генерация и публикация отчёта:
- Скачивает артефакт с результатами
- Генерирует HTML-отчёт с историей запусков
- Публикует отчёт на GitHub Pages

Джоба `reports` выполняется **всегда**, даже если тесты упали — чтобы отчёт об ошибках был доступен.
